# Eval run — 2026-07-10 (S7: structured-response-contract)

First run of S7, testing the newly added Structured Response Contract + templates +
phased-delivery + DevSecOps references (commit 4d547f6). N=1 per arm. **Directional,
not statistical.**

## Method
- **Treatment**: sonnet agent instructed to Read the installed SKILL.md from disk and
  follow it, routing to references itself. It was NOT told any grading criteria. It chose
  to load 5 relevant references (phased-delivery, devsecops, dotnet, architecture,
  review-templates) and skipped the other 8 — routing behaved as designed.
- **Control**: sonnet agent as "senior backend consultant", same task, no skill content.
- **Judge**: a separate opus agent, blind — neutral filenames (brief-K/brief-M, K=treatment,
  M=control), no information about how either was produced, scored strictly against the
  S7 rubric with quoted evidence per cell.

## Results (0–2 per metric)

| metric | control (M) | treatment (K) | delta |
|---|---|---|---|
| decision-structure | 1 | 2 | +1 |
| rejected-alternatives | 1 | 2 | +1 |
| phased-landing | 1 | 2 | +1 |
| grounded-honesty | 1 | 2 | +1 |
| security-governance-depth | 1 | 2 | +1 |
| **total** | **5** | **10** | **+5** |

Judge's adoption verdict: use K as the backbone ("decision-ready: every open question has
an owner, every phase has a gate and a rollback, the exec one-pager is ready to hand over"),
mine M for specific techniques.

## Findings
- **Sonnet discriminated this time.** The 2026-07-09 runs showed a ceiling on sonnet for
  judgment-trap scenarios (S1/S2/S5); S7 shows the contract layer adds measurable value on
  sonnet itself — the delta comes from STRUCTURE and GOVERNANCE (rejected-alternatives as a
  first-class section, per-phase exit criteria, role-named deciders, suppression/baseline
  governance, a filled exec one-pager), not from raw technical knowledge.
- **Control was technically strong** — in places more specific than treatment (explicit
  GitHub OIDC↔Azure AD workload identity federation, CDC/Debezium, canary percentages).
  The skill's contribution was turning comparable technical competence into a governable,
  trackable, decision-ready document. This matches the 2026-07-09 takeaway: the skill's
  value lives where no hard signal forces correctness — here, document completeness and
  decision governance.
- **Honesty rules visibly fired in treatment**: assumptions declared up front, deciders
  named as roles (never invented people), cost figures marked "draft — pending Phase 0"
  instead of fabricated.
- **One coverage miss in treatment**: the break-glass path (present in the pipeline
  template) did not surface in the answer's pipeline section.

## Caveats
- N=1 per arm; one scenario; re-run with N≥5 before quoting the delta.
- A pilot judge run leaked arm identity through filenames (skill-test-control.md) and was
  discarded for the headline result; directionally it agreed (control 7, treatment 10 —
  its stricter/looser control scoring vs the clean judge's 5 also shows judge variance of
  ±2 on the control arm, another reason not to overquote the exact delta).
- Both arms are consulting-style prompts with no repo; S7 tests document/decision quality,
  not repo execution.
- Judge blindness is label-level: a judge can still infer arm identity from style. Full
  blindness would need many mixed answers.
